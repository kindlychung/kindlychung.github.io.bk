Parametric Functions From Different Perspecitives (Dimensions)

May 20th, 2014

Some interesting curves in the 3D space.

sine curve in a circle, sage code
Double click to toggle code 
parametric_plot3d((cos(t), sin(t), 0.2*sin(15*t)), (t, 0, 6.28), aspect_ratio=(1, 1, 1), plot_points=200)

This can be parametrized like this:

\begin{align*} \begin{pmatrix} r \\ \theta \\ z \\ \end{pmatrix} &= \begin{pmatrix} 1 \\ t \\ \sin kt \\ \end{pmatrix} \end{align*}

Translated into xyz-coordinates:

\begin{align*} \begin{pmatrix} \cos t \\ \sin t \\ \sin kt \\ \end{pmatrix} \end{align*}

where t is the angle turned on the xy-plane.

spring in a ring, sage code
Double click to toggle code 
parametric_plot3d(((5+cos(15*t))*cos(t), (5+cos(15*t))*sin(t), sin(15*t)), (t, 0, 6.28), aspect_ratio=(1, 1, 1), plot_points=200)

This curve is more complex, but can be simplified by using a different coordinate system.

  • $r_1$: radius of the torus
  • $\theta_1$: angle on the xy-plane
  • $r_2$: radius of the tube of the torus
  • $\theta_2$: angle on the intersection of the torus

Here is the lateral view. $\theta_1$ is invisible here.

Then we can parametrize it this way:

\begin{align*} \begin{pmatrix} r_1 \\ \theta_1 \\ r_2 \\ \theta_2 \\ \end{pmatrix} &= \begin{pmatrix} r_1 \\ t \\ r_2 \\ kt \\ \end{pmatrix} \end{align*}

Translated into xyz-coordinates:

\begin{align*} \begin{pmatrix} (r1 + \cos kt)\cos t \\ (r1 + \cos kt)\sin t \\ \sin kt \\ \end{pmatrix} \end{align*}

where t is the angle turned on the xy-plane.

Eclipse With Pydev Not Satisfactory

May 20th, 2014

Pydev test, fairly good, but not perfect

I am trying out the pydev plugin for eclipse, so far the code completion looks ok, but the python console is not reliable at all. Here is a very simple test:

Double click to toggle code 
#!/usr/bin/python3

from subprocess import Popen
from subprocess import PIPE
shell_cmd = ["xsel", "-b"]
p = Popen(shell_cmd, stdin=PIPE)
p.communicate("what the hell is this?".encode("ascii"))

This code is just trying to put some text into the system clipboard, and it runs ok in ipython3, yet results are weird in eclipse. The same occurs to pycharm, I have no clue why this is happening.

In the end I find kdevelop with the kdev-python plugin very nice to play with, responsive, snappy, and comes with a konsole terminal on the bottom panel, so that you can paste code directly into an ipython session. Compared to this, eclipse is really just a piece of junk…
